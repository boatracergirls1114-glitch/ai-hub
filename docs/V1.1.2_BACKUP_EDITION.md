# AI Hub v1.1.2 Backup Edition 実装ログ

## 概要

AI Hub v1.1.2 Backup Edition は、AI Hubを安全に運用するためのバックアップ基盤を追加したバージョンです。

v1.1.1 Hotfix では検索・Markdown・コード表示を改善しました。  
v1.1.2 では、実運用で最も重要な「バックアップ」を追加します。

---

## 実装内容

### ローカルバックアップ

AI Hubサーバー内にバックアップを作成します。

```text
/opt/ai-hub/backups
```

保存されるもの

```text
AI Hub本体
PostgreSQLデータベース
復元メモ
```

---

### NASバックアップ

NASがLXCへマウントされている場合、自動でNASへコピーします。

保存先

```text
/mnt/aihub-nas-backup
```

Windows側では以下に対応します。

```text
O:\005_AI分析\AI Hub\backup
```

---

## バックアップファイル

作成されるファイル例

```text
ai-hub-v1.1.2-backup-20260705-181500-files.tar.gz
ai-hub-v1.1.2-backup-20260705-181500-db.sql
ai-hub-v1.1.2-backup-20260705-181500-restore.txt
```

---

## 実行方法

```bash
cd /opt/ai-hub
chmod +x scripts/backup.sh
./scripts/backup.sh
```

## NAS連携

今回の環境では以下の構成でNASバックアップを実現しました。

```text
Synology NAS
  ↓ CIFS
Proxmox Host
  ↓ mp0
LXC105 ai-hub
  ↓ symlink
/mnt/aihub-nas-backup
```

## 発生したトラブル

### 非特権LXCからCIFSを直接マウントできない

エラー

```text
mount error(1): Operation not permitted
```

原因

非特権LXCでは `mount -t cifs` が制限されているため。

対応

Proxmoxホスト側でCIFSマウントし、LXCへ `mp0` で渡した。

---

### 105.conf の改行崩れ

現象

```text
unable to parse value of 'net0'
missing 'ostype'
```

原因

`105.conf` の `net0`・`ostype`・`rootfs` が1行に連結された。

対応

`105.conf` を正常な行構成に戻して復旧。

---

### 書き込み権限エラー

エラー

```text
Permission denied
```

原因

非特権LXCのrootはホスト側ではUID 100000として扱われるため、CIFS側の所有者と合わなかった。

対応

ProxmoxホストでCIFSを以下のように再マウントした。

```bash
mount -t cifs "//192.168.14.140/db-miyo-2f-01" \
/mnt/aihub-backup \
-o credentials=/root/.nascred,vers=3.0,iocharset=utf8,uid=100000,gid=100000,file_mode=0775,dir_mode=0775
```

## 得られた知見

バックアップはAI Hubの実運用において最重要機能です。

GitHubはソースコードの正本として使い、NASはバックアップ保存先として使う構成が安全です。

また、Proxmox LXCでNASを扱う場合、非特権LXCから直接CIFSをマウントするよりも、ProxmoxホストでマウントしてLXCへ渡す方式の方が安定します。
